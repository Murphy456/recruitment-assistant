## 1. 产品概述

### 1.1 产品定位

招聘助手是一款Chrome浏览器插件，帮助HR和猎头在招聘网站上**半自动化**筛选符合JD画像的候选人，并批量打招呼获取简历。

### 1.2 核心价值

- **人机协作模式**：用户始终在"驾驶位"，AI作为"副驾驶"辅助决策
- **零风险设计**：拟人化操作策略，避免触发平台风控
- **效率提升**：列表页粗筛 + 详情页深度分析，大幅减少人工筛选时间

### 1.3 商业模式

- **免费使用**
- 用户自行配置AI API Key
- 离线安装，朋友内测阶段

## 4. 功能需求

### 4.1 JD画像管理

#### 功能描述
用户可创建、编辑、切换多个JD画像，每个JD包含职位要求和评分规则。

#### 功能点

| 功能 | 描述 |
|------|------|
| **创建JD** | 粘贴JD文本 → AI自动解析提取结构化要求 |
| **编辑JD** | 可调整学历、年限、技能、加分项等 |
| **评分规则** | 简单模式（严格程度）/ 高级模式（权重调整） |
| **话术模板** | 每个JD绑定一个打招呼话术模板 |
| **快捷切换** | 顶部下拉菜单快速切换当前JD |
| **JD库管理** | 查看、编辑、删除已保存的JD画像 |

#### 数据结构

```
JD画像
├── 基础信息
│   ├── 名称
│   ├── JD全文
│   └── 创建时间
├── 硬性条件
│   ├── 学历要求
│   ├── 年限要求
│   ├── 专业要求
│   └── 薪资范围
├── 技能要求
│   └── 技能列表
├── 加分项
│   └── 加分标签列表
├── 排除条件
│   └── 过滤条件列表
├── 评分规则
│   ├── 模式（简单/高级）
│   └── 权重配置
└── 话术模板
    └── 带变量的模板文本
```

### 4.2 简历筛选

#### 4.2.1 数据获取方式

| 方式 | 描述 | 优先级 |
|------|------|--------|
| **API响应监听** | 监听平台API响应，直接获取JSON结构化数据 | 优先使用 |
| **DOM解析** | 通过CSS选择器提取页面元素 | 兜底方案 |
| **AI提取** | 将HTML传给AI解析 | 最后兜底 |

**API响应监听优势：**
- 数据结构稳定，不受页面改版影响
- 获取速度更快，无需等待DOM渲染
- 数据更完整，包含页面未展示的信息

#### 4.2.2 列表页粗筛

| 功能 | 描述 |
|------|------|
| **自动触发** | 页面加载后自动分析可见候选人 |
| **滚动触发** | 滚动加载更多候选人时继续分析 |
| **粗筛逻辑** | 基于基础信息（学历、年限、职位）规则匹配 |
| **分数展示** | 每个候选人卡片显示分数和标签 |
| **快速操作** | 一键加入发送队列、跳过、查看详情 |

#### 4.2.2 详情页深度分析

| 功能 | 描述 |
|------|------|
| **简历提取** | CSS选择器 + AI兜底提取完整简历 |
| **数据脱敏** | 发送AI前隐藏手机号、邮箱等隐私信息 |
| **AI匹配** | 调用云端API进行深度匹配分析 |
| **结果呈现** | 分数、标签、推荐理由、顾虑点、面试问题建议 |
| **用户操作** | 加入队列、跳过、标记喜欢/不喜欢 |

#### 匹配结果展示

```
匹配结果面板
├── 综合评分 (0-100)
├── 硬性条件匹配
│   ├── 学历匹配状态
│   ├── 年限匹配状态
│   └── 专业匹配状态
├── 工作经历匹配
│   ├── 匹配分数
│   ├── 亮点列表
│   └── 差距列表
├── 技能匹配
│   ├── 匹配分数
│   ├── 已匹配技能
│   └── 缺失技能
├── 标签
├── 推荐理由
├── 顾虑点
└── 面试问题建议
```

### 4.3 打招呼发送

#### 4.3.1 话术生成

| 功能 | 描述 |
|------|------|
| **模板变量** | 支持{姓名}、{职位}、{公司}、{亮点}等变量 |
| **AI微调** | 根据候选人特点个性化调整措辞 |
| **字数限制** | 适应平台限制（智联200字、BOSS140字） |
| **生成时机** | 加入队列时预生成 |

**AI生成打招呼语逻辑：**

```
输入信息：
- 招聘者信息（姓名、职位、公司）
- 候选人信息（姓名、当前职位、亮点）
- 职位信息（职位名称、关键词）
- 用户个人介绍
- 默认打招呼语模板

AI处理：
1. 分析候选人与职位的匹配点
2. 提取候选人亮点
3. 生成个性化打招呼语（不超过60字）
4. 如果不匹配，返回false使用默认模板

输出：
- 个性化打招呼语
- 或默认模板（当AI返回false时）
```

#### 4.3.2 发送队列

| 功能 | 描述 |
|------|------|
| **队列管理** | 查看、移除、调整待发送候选人 |
| **批量操作** | 全选、反选、一键发送全部 |
| **发送状态** | 待发送、发送中、已发送、发送失败 |

#### 4.3.3 安全发送策略

| 策略 | 配置 |
|------|------|
| **随机延迟** | 30-120秒随机间隔 |
| **每日上限** | 可配置每日发送上限（默认50条） |
| **失败重试** | 拟人化重试，最多5次 |
| **登录检测** | 检测登录态过期，提示用户 |

## 5. 非功能需求

### 5.1 性能要求

| 指标 | 要求 |
|------|------|
| 列表页粗筛响应 | < 100ms（本地规则计算） |
| 详情页AI分析 | < 3s（取决于API响应） |
| 批量分析并发 | 最多3个并发请求 |

### 5.2 安全要求

| 要求 | 描述 |
|------|------|
| 数据脱敏 | 发送AI前隐藏隐私信息 |
| 本地存储 | 所有数据存储在用户本地 |
| API Key保护 | 不上传用户API Key |

### 5.3 兼容性

| 项目 | 要求 |
|------|------|
| 浏览器 | Chrome（最新两个主版本） |
| 平台 | 智联招聘（MVP）、BOSS直聘（后续） |

## 6. 边界场景处理

| 场景 | 处理策略 |
|------|---------|
| 简历信息不完整 | 降低匹配分数，标注缺失信息 |
| JD画像模糊 | AI追问用户澄清需求 |
| 匹配结果为空 | 先自动放宽条件，仍为空则提示用户调整画像 |
| 打招呼发送失败 | 拟人化重试（最多5次），失败后提示用户 |
| 平台登录态过期 | 检测后提示用户重新登录 |
| AI API调用失败 | 显示错误信息，支持重试 |
