## 阶段1：First Principles Thinking 执行结果

### 核心洞察

| 原始假设 | 第一性原理突破 |
|---------|---------------|
| "自动化筛选候选人" | → "人机协作"模式，零风险前提下最大化效率 |
| "在平台上筛选" | → 浏览器插件形态，在用户侧运行 |
| "AI匹配" | → 云端API + 用户反馈学习 + 智能阈值 |
| "自动打招呼" | → 安全队列 + 拟人化发送策略 |

### 最终决策汇总

| 维度 | 决策 |
|------|------|
| **产品形态** | 浏览器插件（Chrome Extension MV3） |
| **平台支持** | 智联招聘(P0) → BOSS直聘(P1) |
| **AI方案** | 云端API（通用多模型支持，用户自配） |
| **简历提取** | CSS选择器 + AI兜底 |
| **匹配深度** | 全维度（基础信息+工作经历+技能+自我评价） |
| **匹配阈值** | 用户反馈学习 → 智能阈值 |
| **打招呼** | 模板+AI微调，安全队列自动发送 |
| **UI形态** | 侧边栏 |
| **数据存储** | 本地IndexedDB，无同步 |
| **账户体系** | 无账户 |
| **商业模式** | 免费，用户自配API Key |
| **数据隐私** | 脱敏处理，只分析公开简历信息 |
| **分发渠道** | 离线安装（朋友内测） |
| **浏览器** | 仅Chrome |
| **性能体验** | 实时显示 + 批量模式 |
| **更新机制** | 本地安装，手动更新 |
| **Prompt模板** | 可配置，用户可自定义 |
| **错误处理** | 导出日志文件 |

### 边界场景处理

| 场景 | 处理策略 |
|------|---------|
| 简历不完整 | 降低分数 |
| JD画像模糊 | AI追问澄清 |
| 匹配为空 | 放宽条件 → 提示调整画像 |
| 发送失败 | 拟人重试(5次) → 提示用户 |
| 登录态过期 | 提示重新登录 |

### 技术栈确认

| 层级 | 技术 |
|------|------|
| 插件框架 | Chrome Extension Manifest V3 |
| UI框架 | React + Tailwind CSS |
| 状态管理 | Zustand |
| DOM解析 | Cheerio |
| 本地存储 | idb (IndexedDB封装) |
| AI接口 | 通用Provider抽象层 |

| **话术模板** | 每JD绑定一个，加入队列时预生成 |
| **历史记录** | 永久保留，今日统计+匹配成功率 |
| **偏好设置** | AI/匹配/发送/显示/隐私五类配置 |
| **用户偏好学习** | 隐式+显式+可查看，设置页+JD编辑展示 |

### 完整技术架构

```
用户界面层: 侧边栏主面板 | 列表页嵌入式UI | 设置页面 | 历史记录页面
业务逻辑层: JD画像管理 | 简历提取 | 匹配引擎 | 话术生成 | 发送队列
平台适配层: 智联招聘适配器 | BOSS直聘适配器
基礎设施层: AI Provider | 本地存储(IndexedDB) | 日志系统 | 用户偏好学习
```
